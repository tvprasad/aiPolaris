# aiPolaris — Use Case Document

Generated by intake prompt — 2026-03-15

## UC-001: Deterministic query execution
**Actor:** Federal operator
**Flow:** Query → Planner (decompose) → Retriever (AI Search) → Synthesizer (cite) → Answer
**Acceptance criteria:**
- Same inputs + pinned model + pinned prompts = same output (replay match ≥ 95%)
- trace_id present on every invocation
- StepRecord appended by every node before returning

## UC-002: Capability-sandboxed tool execution
**Actor:** LangGraph agent node
**Flow:** Node attempts tool call → manifest check → execute or raise CapabilityViolationError
**Acceptance criteria:**
- CapabilityViolationError raised and logged on any manifest violation
- Zero write operations without explicit RBAC gate
- Test deliberately triggers the error and passes

## UC-003: Conditional branching on retrieval confidence
**Actor:** System (automatic)
**Flow:** Retriever returns chunks → confidence check → synthesize or refusal path
**Acceptance criteria:**
- Correct refusal rate: 100% on out-of-scope questions
- Incorrect refusal rate: < 10% on in-scope questions
- Refusal path logged with trace_id and category

## UC-004: Re-playable audit from trace_id
**Actor:** Compliance officer / ISSO
**Flow:** trace_id lookup → full StepRecord sequence → reconstruct execution
**Acceptance criteria:**
- Every trace_id produces complete StepRecord sequence
- Replay produces identical output given pinned model + prompts

## UC-005: GCCH deployment without code changes
**Actor:** DevOps / deployment team
**Flow:** terraform workspace select gcch → terraform apply → all endpoints switch
**Acceptance criteria:**
- terraform plan env=gcch produces valid plan
- All endpoints resolve to *.azure.us equivalents
- Zero hardcoded commercial URLs in application code
